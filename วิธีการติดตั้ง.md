# วิธี Upload Website ขึ้น Netlify ผ่าน GitHub

คู่มือนี้อธิบายวิธีนำเว็บไซต์ในโฟลเดอร์ `D:\webapp\tinmeaw-manager` ขึ้นออนไลน์ด้วย GitHub และ Netlify แบบละเอียด เหมาะสำหรับเว็บ static ที่มีไฟล์ `index.html` อยู่ที่ root ของโปรเจกต์

โปรเจกต์นี้มีไฟล์ `netlify.toml` แล้ว:

```toml
[build]
  publish = "."
```

ดังนั้นตอนตั้งค่าบน Netlify ให้ใช้ค่าเหล่านี้:

```txt
Build command: เว้นว่าง
Publish directory: .
Base directory: เว้นว่าง
Production branch: main
```

## ภาพรวมขั้นตอน

1. ตรวจเว็บในเครื่องก่อน
2. ส่งโค้ดขึ้น GitHub
3. เชื่อม GitHub กับ Netlify
4. ตั้งค่า build และ deploy
5. ตรวจเว็บหลัง deploy
6. ตั้งค่า custom domain ถ้ามีโดเมนของตัวเอง

## สิ่งที่ต้องมีก่อนเริ่ม

- บัญชี GitHub: https://github.com
- บัญชี Netlify: https://app.netlify.com
- ติดตั้ง Git แล้ว: https://git-scm.com/downloads
- มีโฟลเดอร์เว็บในเครื่อง เช่น `D:\webapp\tinmeaw-manager`

ตรวจว่า Git ใช้งานได้:

```powershell
git --version
```

เข้าโฟลเดอร์โปรเจกต์:

```powershell
cd D:\webapp\tinmeaw-manager
```

ตรวจไฟล์สำคัญ:

```powershell
dir
```

ควรเห็นไฟล์หลัก เช่น:

```txt
index.html
privacy.html
terms.html
robots.txt
_headers
netlify.toml
css\
js\
```

## ขั้นตอนที่ 1: ตรวจเว็บก่อน upload

เปิดไฟล์นี้ใน browser:

```txt
D:\webapp\tinmeaw-manager\index.html
```

ให้เช็กก่อนว่า:

- หน้าแรกเปิดได้
- CSS แสดงถูกต้อง
- ปุ่มและเมนูหลักใช้งานได้
- ลิงก์ไป `privacy.html` และ `terms.html` เปิดได้
- ไม่มีไฟล์ลับ เช่น `.env`, service account, private key, client secret

ข้อควรระวัง:

- ห้าม commit Firebase service account ลง GitHub
- ห้าม commit Google OAuth Client Secret ลง frontend
- ถ้ามีค่า config ที่เป็น public frontend config ให้ใช้เฉพาะค่าที่เปิดเผยได้เท่านั้น

## ขั้นตอนที่ 2: สร้าง Repository บน GitHub

1. เข้า https://github.com
2. กดปุ่ม `New repository`
3. ตั้งชื่อ repo เช่น:

```txt
tinmeaw-manager
```

4. เลือก `Public` หรือ `Private`
5. ถ้าโปรเจกต์มีไฟล์อยู่ในเครื่องแล้ว ให้ไม่ต้องติ๊ก:

```txt
Add a README file
Add .gitignore
Choose a license
```

6. กด `Create repository`
7. คัดลอก URL ของ repo เช่น:

```txt
https://github.com/USERNAME/tinmeaw-manager.git
```

ให้เปลี่ยน `USERNAME` เป็นชื่อบัญชี GitHub ของคุณ

## ขั้นตอนที่ 3: ส่งโค้ดจากเครื่องขึ้น GitHub

เปิด PowerShell แล้วเข้าโฟลเดอร์โปรเจกต์:

```powershell
cd D:\webapp\tinmeaw-manager
```

ตรวจสถานะ Git:

```powershell
git status
```

ถ้าเจอ error ประมาณนี้:

```txt
fatal: detected dubious ownership in repository
```

ให้เพิ่ม safe directory:

```powershell
git config --global --add safe.directory D:/webapp/tinmeaw-manager
```

แล้วลองอีกครั้ง:

```powershell
git status
```

ถ้าโฟลเดอร์ยังไม่ได้เป็น Git repository ให้เริ่มด้วย:

```powershell
git init
```

ตั้งชื่อ branch เป็น `main`:

```powershell
git branch -M main
```

เพิ่มไฟล์ทั้งหมดเข้า Git:

```powershell
git add .
```

commit ไฟล์:

```powershell
git commit -m "Initial website deploy"
```

เพิ่ม remote GitHub:

```powershell
git remote add origin https://github.com/USERNAME/tinmeaw-manager.git
```

ถ้ามี remote อยู่แล้ว และต้องการเปลี่ยน URL:

```powershell
git remote set-url origin https://github.com/USERNAME/tinmeaw-manager.git
```

ตรวจ remote:

```powershell
git remote -v
```

push ขึ้น GitHub:

```powershell
git push -u origin main
```

ถ้า GitHub ขอ login ให้ login ตามขั้นตอนที่ Git แสดง หรือใช้ GitHub Desktop แทนได้

## วิธีทางเลือก: ใช้ GitHub Desktop

ถ้าไม่ถนัด command line:

1. ติดตั้ง GitHub Desktop: https://desktop.github.com
2. Login ด้วยบัญชี GitHub
3. เลือก `File > Add local repository`
4. เลือกโฟลเดอร์:

```txt
D:\webapp\tinmeaw-manager
```

5. ถ้ายังไม่เป็น Git repository ให้ GitHub Desktop จะมีปุ่มให้สร้าง repository
6. ใส่ commit message เช่น:

```txt
Initial website deploy
```

7. กด `Commit to main`
8. กด `Publish repository`
9. เลือกว่าจะให้ repo เป็น `Private` หรือ `Public`
10. กด publish แล้วรอ upload เสร็จ

## ขั้นตอนที่ 4: เชื่อม GitHub กับ Netlify

1. เข้า https://app.netlify.com
2. เลือก team หรือ account ที่ต้องการ deploy
3. กด `Add new project`
4. เลือก `Import an existing project`
5. เลือก Git provider เป็น `GitHub`
6. Netlify จะขอ authorization เพื่อเข้าถึง GitHub
7. ถ้ามีหน้าให้ติดตั้ง Netlify GitHub App ให้เลือก repo ที่ต้องการ เช่น:

```txt
tinmeaw-manager
```

8. เลือก repository จากรายการ

## ขั้นตอนที่ 5: ตั้งค่า Deploy บน Netlify

ในหน้า build settings ให้ใช้ค่านี้สำหรับโปรเจกต์นี้:

```txt
Branch to deploy: main
Base directory: เว้นว่าง
Build command: เว้นว่าง
Publish directory: .
```

เหตุผลที่ `Publish directory` เป็น `.` เพราะไฟล์ `index.html` อยู่ที่ root ของโปรเจกต์ และ `netlify.toml` กำหนดไว้แล้วว่า:

```toml
[build]
  publish = "."
```

จากนั้นกด:

```txt
Publish
```

หรือถ้าหน้า Netlify ใช้คำว่า deploy ให้กดปุ่ม deploy ตามที่หน้าเว็บแสดง

## ขั้นตอนที่ 6: ตรวจผล Deploy

หลังจากกด publish แล้ว ให้รอ Netlify build และ deploy

ให้ตรวจในหน้า deploy log:

- ไม่มี error สีแดง
- deploy สำเร็จ
- มี URL ประมาณนี้:

```txt
https://your-site-name.netlify.app
```

เปิด URL แล้วตรวจ:

- หน้าแรกโหลดได้
- CSS และ JavaScript โหลดครบ
- หน้า privacy และ terms เปิดได้
- login หรือ workflow หลักทำงานตามที่คาดไว้
- browser console ไม่มี error สำคัญ

## ขั้นตอนที่ 7: การ update เว็บรอบต่อไป

หลังจากเชื่อม GitHub กับ Netlify แล้ว ทุกครั้งที่แก้เว็บ ให้ทำตามนี้:

```powershell
cd D:\webapp\tinmeaw-manager
git status
git add .
git commit -m "Update website"
git push
```

เมื่อ push สำเร็จ Netlify จะ deploy ใหม่ให้อัตโนมัติ

ตรวจ deploy ได้ที่:

```txt
Netlify project > Deploys
```

## ขั้นตอนที่ 8: ตั้งชื่อเว็บ Netlify

Netlify จะสร้างชื่อเว็บสุ่มให้ เช่น:

```txt
https://random-name-12345.netlify.app
```

ถ้าต้องการเปลี่ยนชื่อ:

1. เข้า project บน Netlify
2. ไปที่ project settings หรือ site details
3. หาเมนูสำหรับเปลี่ยน site name
4. เปลี่ยนเป็นชื่อที่ต้องการ เช่น:

```txt
tinmeaw-manager
```

หลังเปลี่ยนแล้ว URL จะเป็น:

```txt
https://tinmeaw-manager.netlify.app
```

ถ้าชื่อนั้นถูกใช้แล้ว ต้องเลือกชื่ออื่น

## ขั้นตอนที่ 9: ตั้งค่า Custom Domain

ถ้ามีโดเมนของตัวเอง เช่น:

```txt
example.com
www.example.com
```

ให้ทำบน Netlify:

1. เข้า project
2. ไปที่ `Domain management`
3. เลือก `Production domains`
4. กด `Add a domain`
5. ใส่โดเมนของคุณ
6. ทำตามคำแนะนำ DNS ที่ Netlify แสดง

ถ้าใช้ subdomain เช่น `www.example.com` ให้ตั้งค่า DNS แบบ CNAME:

```txt
Type: CNAME
Name/Host: www
Value/Target: your-site-name.netlify.app
```

ถ้าใช้ apex domain เช่น `example.com` ให้ทำตามค่าที่ Netlify แสดงใน `Pending DNS verification`

โดยทั่วไป Netlify แนะนำ:

```txt
Type: ALIAS / ANAME / flattened CNAME
Name/Host: @
Value/Target: apex-loadbalancer.netlify.com
```

ถ้า DNS provider ไม่รองรับ ALIAS, ANAME หรือ flattened CNAME อาจต้องใช้ A record:

```txt
Type: A
Name/Host: @
Value/Target: 75.2.60.5
```

หลังแก้ DNS อาจต้องรอหลายชั่วโมง และบางครั้งอาจนานถึง 24 ชั่วโมง

## ขั้นตอนที่ 10: ตรวจ HTTPS

หลังตั้ง custom domain แล้ว Netlify จะพยายามออก SSL certificate ให้อัตโนมัติ

ตรวจได้ที่:

```txt
Netlify project > Domain management > HTTPS
```

ถ้า HTTPS ยังไม่ผ่าน ให้ตรวจ:

- DNS record ชี้มาที่ Netlify ถูกต้องหรือไม่
- `www` เป็น CNAME ไปที่ `your-site-name.netlify.app` หรือไม่
- apex domain ชี้ตามค่าที่ Netlify แนะนำหรือไม่
- รอ DNS propagation ครบหรือยัง

## ปัญหาที่พบบ่อย

### 1. Deploy แล้วขึ้น Page not found

สาเหตุที่พบบ่อย:

- `Publish directory` ตั้งผิด
- ไม่มี `index.html` ใน publish directory

สำหรับโปรเจกต์นี้ให้ใช้:

```txt
Publish directory: .
```

### 2. CSS หรือ JavaScript ไม่โหลด

ตรวจว่า path ใน HTML เป็น path แบบ relative เช่น:

```html
<link rel="stylesheet" href="css/style.css">
<script src="js/app.js"></script>
```

ถ้าใช้ path ที่ขึ้นต้นด้วย `/` ต้องแน่ใจว่าไฟล์อยู่ตรงตำแหน่งนั้นจริงเมื่อ deploy

### 3. Push ไม่ได้เพราะ authentication

ให้ลอง:

- Login GitHub ใน browser ก่อน
- ใช้ GitHub Desktop
- ใช้ Git Credential Manager
- ใช้ personal access token เฉพาะเมื่อจำเป็น และห้ามเก็บ token ไว้ในไฟล์โปรเจกต์

### 4. Netlify หา repository ไม่เจอ

ให้ตรวจ:

- Netlify GitHub App มีสิทธิ์เข้าถึง repo แล้วหรือยัง
- Repo อยู่ใน account หรือ organization ที่เลือกถูกต้องหรือไม่
- ถ้า repo เป็น private ต้องให้สิทธิ์ Netlify เข้าถึง private repo

### 5. แก้ไฟล์แล้วเว็บไม่ update

ให้ตรวจ:

```powershell
git status
git log --oneline -5
git remote -v
git push
```

แล้วดูใน Netlify:

```txt
Project > Deploys
```

ถ้าไม่มี deploy ใหม่ แปลว่า push อาจยังไม่ถึง GitHub หรือ Netlify ยังไม่ได้เชื่อมกับ branch ที่ถูกต้อง

- [ ] Build command เว้นว่าง
- [ ] Publish directory เป็น `.`
- [ ] เปิดเว็บผ่าน `netlify.app` ได้

- [ ] HTTPS ใช้งานได้
- [ ] ไม่มี secret หรือ private key ใน repo
- [ ] ทดสอบ workflow หลักแล้ว
