# วิธี Upload Website ขึ้น Netlify ผ่าน GitHub

คู่มือนี้อธิบายวิธีนำเว็บไซต์ในโฟลเดอร์ `D:\webapp\tinmeaw-manager` ขึ้นออนไลน์ด้วย GitHub และ Netlify แบบละเอียด เหมาะสำหรับเว็บ static ที่มีไฟล์ `index.html` อยู่ที่ root ของโปรเจกต์

โปรเจกต์นี้มีไฟล์ `netlify.toml` แล้ว:

```toml
[build]
  publish = "."
```

ดังนั้นตอนตั้งค่าบน Netlify ให้ใช้ค่าเหล่านี้:

```txt
Build command: เว้นว่าง
Publish directory: .
Base directory: เว้นว่าง
Production branch: main
```

## ภาพรวมขั้นตอน

1. ตรวจเว็บในเครื่องก่อน
2. ส่งโค้ดขึ้น GitHub
3. เชื่อม GitHub กับ Netlify
4. ตั้งค่า build และ deploy
5. ตรวจเว็บหลัง deploy
6. ตั้งค่า Firebase สำหรับ Login และฐานข้อมูล
7. ตั้งค่า Google Cloud สำหรับ Sheets และ Calendar
8. ตั้งค่า custom domain ถ้ามีโดเมนของตัวเอง

- บัญชี GitHub: https://github.com
- บัญชี Netlify: https://app.netlify.com
- บัญชี Google สำหรับ Firebase และ Google Cloud
- ติดตั้ง Git แล้ว: https://git-scm.com/downloads
- มีโฟลเดอร์เว็บในเครื่อง เช่น `D:\webapp\tinmeaw-manager`

## ขั้นตอนที่ 1: ตรวจเว็บก่อน upload

เปิดไฟล์นี้ใน browser:

```txt
D:\webapp\tinmeaw-manager\index.html
```

ให้เช็กก่อนว่า:

- หน้าแรกเปิดได้
- CSS แสดงถูกต้อง
- ปุ่มและเมนูหลักใช้งานได้
- ลิงก์ไป `privacy.html` และ `terms.html` เปิดได้
- ไม่มีไฟล์ลับ เช่น `.env`, service account, private key, client secret

ข้อควรระวัง:

- ห้าม commit Firebase service account ลง GitHub
- ห้าม commit Google OAuth Client Secret ลง frontend
- ถ้ามีค่า config ที่เป็น public frontend config ให้ใช้เฉพาะค่าที่เปิดเผยได้เท่านั้น

## ขั้นตอนที่ 2: สร้าง Repository บน GitHub

1. เข้า https://github.com
2. กดปุ่ม `New repository`
3. ตั้งชื่อ repo เช่น:

```txt
tinmeaw-manager
```

4. เลือก `Public` หรือ `Private`
5. ถ้าโปรเจกต์มีไฟล์อยู่ในเครื่องแล้ว ให้ไม่ต้องติ๊ก:

```txt
Add a README file
Add .gitignore
Choose a license
```

6. กด `Create repository`
7. คัดลอก URL ของ repo เช่น:

```txt
https://github.com/USERNAME/tinmeaw-manager.git
```

ให้เปลี่ยน `USERNAME` เป็นชื่อบัญชี GitHub ของคุณ

## ขั้นตอนที่ 3: ส่งโค้ดจากเครื่องขึ้น GitHub

เปิด PowerShell แล้วเข้าโฟลเดอร์โปรเจกต์:

```powershell
cd D:\webapp\tinmeaw-manager
```

ตรวจสถานะ Git:

```powershell
git status
```

ถ้าเจอ error ประมาณนี้:

```txt
fatal: detected dubious ownership in repository
```

ให้เพิ่ม safe directory:

```powershell
git config --global --add safe.directory D:/webapp/tinmeaw-manager
```

แล้วลองอีกครั้ง:

```powershell
git status
```

ถ้าโฟลเดอร์ยังไม่ได้เป็น Git repository ให้เริ่มด้วย:

```powershell
git init
```

ตั้งชื่อ branch เป็น `main`:

```powershell
git branch -M main
```

เพิ่มไฟล์ทั้งหมดเข้า Git:

```powershell
git add .
```

commit ไฟล์:

```powershell
git commit -m "Initial website deploy"
```

เพิ่ม remote GitHub:

```powershell
git remote add origin https://github.com/USERNAME/tinmeaw-manager.git
```

ถ้ามี remote อยู่แล้ว และต้องการเปลี่ยน URL:

```powershell
git remote set-url origin https://github.com/USERNAME/tinmeaw-manager.git
```

ตรวจ remote:

```powershell
git remote -v
```

push ขึ้น GitHub:

```powershell
git push -u origin main
```

ถ้า GitHub ขอ login ให้ login ตามขั้นตอนที่ Git แสดง หรือใช้ GitHub Desktop แทนได้

ถ้าไม่ถนัด command line:

1. ติดตั้ง GitHub Desktop: https://desktop.github.com
2. Login ด้วยบัญชี GitHub
3. เลือก `File > Add local repository`
4. เลือกโฟลเดอร์:

```txt

```

5. ถ้ายังไม่เป็น Git repository ให้ GitHub Desktop จะมีปุ่มให้สร้าง repository

## ขั้นตอนที่ 4: เชื่อม GitHub กับ Netlify

1. เข้า https://app.netlify.com
2. เลือก team หรือ account ที่ต้องการ deploy
3. กด `Add new project`
4. เลือก `Import an existing project`
5. เลือก Git provider เป็น `GitHub`
6. Netlify จะขอ authorization เพื่อเข้าถึง GitHub
7. ถ้ามีหน้าให้ติดตั้ง Netlify GitHub App ให้เลือก repo ที่ต้องการ เช่น:

```txt
tinmeaw-manager
```

8. เลือก repository จากรายการ

## ขั้นตอนที่ 5: ตั้งค่า Deploy บน Netlify

ในหน้า build settings ให้ใช้ค่านี้สำหรับโปรเจกต์นี้:

```txt
Branch to deploy: main
Base directory: เว้นว่าง
Build command: เว้นว่าง
Publish directory: .
```

เหตุผลที่ `Publish directory` เป็น `.` เพราะไฟล์ `index.html` อยู่ที่ root ของโปรเจกต์ และ `netlify.toml` กำหนดไว้แล้วว่า:

```toml
[build]
  publish = "."
```

จากนั้นกด:

```txt
Publish
```

หรือถ้าหน้า Netlify ใช้คำว่า deploy ให้กดปุ่ม deploy ตามที่หน้าเว็บแสดง

## ขั้นตอนที่ 6: ตรวจผล Deploy

หลังจากกด publish แล้ว ให้รอ Netlify build และ deploy

## ขั้นตอนที่ 9: ติดตั้งและตั้งค่า Firebase

ในคู่มือนี้คำว่า `Filebase` หมายถึง `Firebase` ซึ่งเป็นบริการของ Google ที่เว็บนี้ใช้สำหรับ Login และเก็บข้อมูล Firestore

สำหรับโปรเจกต์นี้ Firebase ใช้ทำ 3 ส่วนหลัก:

- Firebase Authentication สำหรับ Login ด้วย Google
- Cloud Firestore สำหรับเก็บข้อมูลงานและการตั้งค่าของผู้ใช้
- Firebase App Check เป็นตัวเลือกเสริมสำหรับลดการเรียกใช้งานจากเว็บปลอม

### 9.1 สร้าง Firebase Project

1. เข้า https://console.firebase.google.com/
2. กด `Add project`
3. ตั้งชื่อ project เช่น:

```txt
tinmeawfoto-manager
```

4. เลือก Google account ที่จะเป็นเจ้าของระบบ
5. ถ้ามีตัวเลือก Google Analytics จะเปิดหรือปิดก็ได้ เว็บนี้ไม่ได้บังคับใช้ Analytics
6. กดสร้าง project แล้วรอให้ Firebase เตรียม project เสร็จ

หลังสร้างเสร็จ ให้จำค่า `Project ID` ไว้ เพราะต้องใช้ตรวจว่า config ในเว็บตรงกับ Firebase project ที่เลือก

### 9.2 เพิ่ม Web App ใน Firebase

1. เข้า Firebase project
2. ไปที่ `Project settings`
3. ในแท็บ `General` หา section `Your apps`
4. กดไอคอน Web app `</>`
5. ตั้งชื่อ app เช่น:

```txt
Tinmeaw Manager Web
```

6. ไม่ต้องเปิด Firebase Hosting เพราะเว็บนี้ deploy ผ่าน Netlify
7. กด `Register app`
8. Firebase จะแสดง `Firebase SDK config`

ให้คัดลอกเฉพาะ object config มาใส่ในไฟล์:

```txt
js/config.js
```

ในส่วนนี้:

```js
FIREBASE_CONFIG: {
  apiKey: 'ใส่ค่าจาก Firebase',
  authDomain: 'ใส่ค่าจาก Firebase',
  projectId: 'ใส่ค่าจาก Firebase',
  storageBucket: 'ใส่ค่าจาก Firebase',
  messagingSenderId: 'ใส่ค่าจาก Firebase',
  appId: 'ใส่ค่าจาก Firebase',
  measurementId: '',
}
```

ค่า Firebase Web config เป็น public identifier ที่ frontend เห็นได้ ไม่ใช่ client secret แต่ต้องป้องกันข้อมูลจริงด้วย Firestore Rules และ Authorized domains

ห้ามใส่ข้อมูลเหล่านี้ใน frontend:

- Firebase service account JSON
- Private key
- Google OAuth Client Secret
- Token ส่วนตัว
- Password หรือ API secret ที่ใช้แทนตัวเจ้าของระบบได้

### 9.3 เปิด Firebase Authentication ด้วย Google

1. เข้า Firebase project
2. ไปที่ `Authentication`
3. กด `Get started` ถ้ายังไม่เคยเปิดใช้
4. ไปที่แท็บ `Sign-in method`
5. เลือก provider `Google`
6. เปิด `Enable`
7. เลือก support email
8. กด `Save`

หลังจาก deploy เว็บขึ้น Netlify แล้ว ให้เพิ่ม domain ที่อนุญาต:

1. ไปที่ `Authentication > Settings`
2. หา `Authorized domains`
3. เพิ่ม domain เหล่านี้ตามที่ใช้จริง:

```txt
localhost
127.0.0.1
your-site-name.netlify.app
yourdomain.com
www.yourdomain.com
```

ถ้า login แล้วเจอ error `auth/unauthorized-domain` แปลว่ายังไม่ได้เพิ่ม domain ของเว็บในจุดนี้

### 9.4 สร้าง Cloud Firestore

1. เข้า Firebase project
2. ไปที่ `Firestore Database`
3. กด `Create database`
4. เลือก `Production mode`
5. เลือก location ของ database ที่ใกล้ผู้ใช้ หรือ location ที่เหมาะกับบัญชี Google Cloud ของคุณ
6. กดสร้าง database

เว็บนี้เก็บข้อมูลภายใต้ path ลักษณะนี้:

```txt
users/{userId}/{email-name}-jobs/{jobId}
users/{userId}/settings/app
```

ดังนั้น Firestore Rules ควรจำกัดให้ผู้ใช้เห็นเฉพาะข้อมูลของตัวเอง

### 9.5 ใส่ Firestore Rules

ไปที่:

```txt
Firebase Console > Firestore Database > Rules
```

วาง rules นี้:

```js
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
  }
}
```

จากนั้นกด `Publish`

ห้ามใช้ rule แบบนี้บน production:

```js
allow read, write: if true;
```

เพราะจะทำให้ทุกคนอ่านและเขียนข้อมูลในฐานข้อมูลได้

### 9.6 ตั้งค่า Firebase App Check (แนะนำ แต่ทำหลังระบบหลักใช้งานได้ก่อน)

ใน `js/config.js` มีช่องนี้:

```js
FIREBASE_APP_CHECK_SITE_KEY: '',
```

ถ้าต้องการเปิด App Check:

1. เข้า Firebase project
2. ไปที่ `App Check`
3. เลือก Web app ของโปรเจกต์นี้
4. เลือก provider เช่น `reCAPTCHA v3` หรือ `reCAPTCHA Enterprise`
5. เพิ่ม domain ที่ใช้จริง เช่น `your-site-name.netlify.app` และ custom domain
6. คัดลอก site key มาใส่ใน `FIREBASE_APP_CHECK_SITE_KEY`
7. ทดสอบ login, อ่าน/เขียน Firestore และ sync ก่อน
8. ค่อยเปิด enforce หลังมั่นใจว่าเว็บทำงานครบ

อย่าเปิด enforce ทันทีถ้ายังไม่ได้ทดสอบ เพราะอาจทำให้เว็บอ่านหรือเขียน Firestore ไม่ได้

## ขั้นตอนที่ 10: ตั้งค่า Google Cloud, Sheets และ Calendar

เว็บนี้ใช้ Google เพิ่มเติมสำหรับ:

- Google Login ผ่าน Firebase Authentication
- Google Sheets สำหรับ sync ข้อมูลงาน
- Google Calendar สำหรับ sync ตารางงาน
- Google Drive scope `drive.file` สำหรับสร้างหรือจัดการไฟล์ Sheet ที่ผู้ใช้อนุญาต

ให้ใช้ Google Cloud project เดียวกับ Firebase project เพื่อให้ Auth, OAuth consent และ API อยู่ชุดเดียวกัน

### 10.1 เปิด Google Cloud project ให้ตรงกับ Firebase

1. เข้า https://console.cloud.google.com/
2. กดตัวเลือก project ด้านบน
3. เลือก project ที่มี `Project ID` ตรงกับ Firebase
4. ถ้าไม่เจอ ให้ตรวจว่า login ด้วย Google account เดียวกับที่สร้าง Firebase

### 10.2 Enable APIs ที่ต้องใช้

ไปที่:

```txt
Google Cloud Console > APIs & Services > Library
```

ค้นหาและกด `Enable` ทีละตัว:

```txt
Google Sheets API
Google Calendar API
Google Drive API
```

ถ้าไม่เปิด API เหล่านี้ ปุ่ม sync อาจขึ้น error ว่า API ยังไม่ถูกเปิดใช้ หรือ permission ไม่พอ

### 10.3 ตั้งค่า Google Auth Platform และ OAuth Consent Screen

ไปที่:

```txt
Google Cloud Console > APIs & Services > Google Auth platform
```

บางบัญชีอาจยังเห็นชื่อเมนูเดิมเป็น `OAuth consent screen`

ตั้งค่าหลัก:

```txt
User type: External
App name: Tinmeaw Manager
User support email: อีเมลของเจ้าของระบบ
Developer contact information: อีเมลของเจ้าของระบบ
```

ในส่วน App domain ให้ใส่ URL หลัง deploy แล้ว:

```txt
Application home page: https://your-site-name.netlify.app/
Application privacy policy link: https://your-site-name.netlify.app/privacy.html
Application terms of service link: https://your-site-name.netlify.app/terms.html
```

ถ้ามี custom domain ให้ใช้ domain จริงแทน:

```txt
Application home page: https://yourdomain.com/
Application privacy policy link: https://yourdomain.com/privacy.html
Application terms of service link: https://yourdomain.com/terms.html
```

เพิ่ม scopes ที่เว็บนี้ใช้:

```txt
openid
email
profile
https://www.googleapis.com/auth/spreadsheets
https://www.googleapis.com/auth/drive.file
https://www.googleapis.com/auth/calendar
```

ถ้า app ยังอยู่ใน Testing mode ให้เพิ่มอีเมลผู้ใช้งานจริงใน `Test users` ด้วย ไม่อย่างนั้นผู้ใช้ที่ไม่ได้อยู่ในรายชื่ออาจ login หรืออนุญาต scope ไม่ได้

ถ้าต้องเปิดให้คนทั่วไปใช้นอกกลุ่ม test users อาจต้อง publish app และบาง scope อาจต้องผ่านขั้นตอนตรวจสอบจาก Google

ตำแหน่งเมนูในหน้าจอใหม่โดยทั่วไปคือ:

```txt
Branding: ตั้งชื่อ app, support email, logo, app domain, privacy, terms
Audience: เลือก Internal/External และเพิ่ม Test users
Data Access: เพิ่มหรือลด OAuth scopes
```

### 10.4 OAuth Client ID สำหรับโปรเจกต์นี้

สำหรับ static deploy บน Netlify แบบปัจจุบัน ให้ใช้ Firebase Authentication Google provider เป็นหลัก ดังนั้นค่าเหล่านี้ใน `js/config.js` ควรปล่อยว่างไว้:

```js
RUNTIME_CONFIG_ENDPOINT: '',
GOOGLE_OAUTH_CLIENT_ID: '',
GOOGLE_OAUTH_TOKEN_ENDPOINT: '',
```

ไม่ต้องใส่ Google OAuth Client Secret ใน frontend เด็ดขาด

ถ้าภายหลังทำ backend สำหรับ OAuth code flow เอง ค่อยสร้าง OAuth Client ID แบบ `Web application` และตั้งค่า:

```txt
Authorized JavaScript origins:
https://your-site-name.netlify.app
https://yourdomain.com

Authorized redirect URIs:
https://your-site-name.netlify.app
https://yourdomain.com
```

แต่ถ้าไม่มี backend สำหรับแลก token ให้ปล่อย `GOOGLE_OAUTH_TOKEN_ENDPOINT` ว่างไว้ตามเดิม

### 10.5 ตั้งค่า Google Sheets

มี 2 วิธี

วิธีที่ 1: ให้เว็บสร้าง Sheet ให้:

1. เปิดเว็บที่ deploy แล้ว
2. Login ด้วย Google
3. ไปที่เมนูตั้งค่า หรือปุ่มเชื่อม Google Sheet ในเว็บ
4. กดสร้าง Sheet ถ้าเว็บมีปุ่มให้สร้าง
5. ระบบจะบันทึก Sheet ID ไว้ใน settings ของผู้ใช้

วิธีที่ 2: ใช้ Sheet ที่สร้างเอง:

1. เข้า https://sheets.google.com/
2. สร้าง Spreadsheet ใหม่
3. ตั้งชื่อ เช่น:

```txt
Tinmeaw Manager Jobs
```

4. สร้างหรือเปลี่ยนชื่อ tab เป็น:

```txt
Jobs
```

5. คัดลอก Sheet ID จาก URL

ตัวอย่าง URL:

```txt
https://docs.google.com/spreadsheets/d/1AbCDefGhijk1234567890/edit
```

Sheet ID คือ:

```txt
1AbCDefGhijk1234567890
```

6. ใส่ Sheet ID ในเมนูตั้งค่าของเว็บ หรือใส่เป็นค่าเริ่มต้นใน `js/config.js`:

```js
SHEET_ID: '1AbCDefGhijk1234567890',
SHEET_NAME: 'Jobs',
```

ถ้า Sheet อยู่ในบัญชี Google คนอื่น ต้อง share ให้บัญชีที่ login ในเว็บมีสิทธิ์แก้ไขก่อน

### 10.6 ตั้งค่า Google Calendar

ค่าเริ่มต้นในโปรเจกต์นี้คือ:

```js
CALENDAR_ID: 'primary',
```

แปลว่า sync ไปยัง calendar หลักของบัญชี Google ที่ login

ถ้าต้องการใช้ calendar เฉพาะ:

1. เข้า https://calendar.google.com/
2. ไปที่ calendar ที่ต้องการ
3. เปิด `Settings and sharing`
4. หา `Calendar ID`
5. นำค่า Calendar ID ไปใส่ในเมนูตั้งค่าของเว็บ หรือใส่ใน `js/config.js`

ตัวอย่าง:

```js
CALENDAR_ID: 'your-calendar-id@group.calendar.google.com',
```

บัญชีที่ login ต้องมีสิทธิ์สร้างและแก้ไข event ใน calendar นั้น

### 10.7 ทดสอบ Google และ Firebase หลังตั้งค่า

หลัง deploy และตั้งค่า Firebase/Google แล้ว ให้ทดสอบตามลำดับ:

1. เปิดเว็บผ่าน `https://your-site-name.netlify.app`
2. กด Login ด้วย Google
3. เพิ่มงานตัวอย่าง 1 งาน
4. refresh หน้าเว็บ แล้วตรวจว่างานยังอยู่ แปลว่า Firestore ทำงาน
5. กด sync Google Sheets แล้วตรวจว่า Sheet มีข้อมูล
6. กด sync Google Calendar แล้วตรวจว่า Calendar มี event
7. Logout แล้ว login ใหม่อีกครั้ง

ถ้าขั้นตอนใดพัง ให้ดูหัวข้อปัญหาที่พบบ่อยด้านล่าง

## ขั้นตอนที่ 11: ตั้งค่า Custom Domain

ถ้ามีโดเมนของตัวเอง เช่น:

```txt
example.com
www.example.com
```

ให้ทำบน Netlify:

1. เข้า project
2. ไปที่ `Domain management`
3. เลือก `Production domains`
4. กด `Add a domain`
5. ใส่โดเมนของคุณ
6. ทำตามคำแนะนำ DNS ที่ Netlify แสดง

ถ้าใช้ subdomain เช่น `www.example.com` ให้ตั้งค่า DNS แบบ CNAME:

```txt
Type: CNAME
Name/Host: www
Value/Target: your-site-name.netlify.app
```

ถ้าใช้ apex domain เช่น `example.com` ให้ทำตามค่าที่ Netlify แสดงใน `Pending DNS verification`

โดยทั่วไป Netlify แนะนำ:

```txt
Type: ALIAS / ANAME / flattened CNAME
Name/Host: @
Value/Target: apex-loadbalancer.netlify.com
```

ถ้า DNS provider ไม่รองรับ ALIAS, ANAME หรือ flattened CNAME อาจต้องใช้ A record:

```txt
Type: A
Name/Host: @
Value/Target: 75.2.60.5
```

หลังแก้ DNS อาจต้องรอหลายชั่วโมง และบางครั้งอาจนานถึง 24 ชั่วโมง

## ขั้นตอนที่ 12: ตรวจ HTTPS

หลังตั้ง custom domain แล้ว Netlify จะพยายามออก SSL certificate ให้อัตโนมัติ

ตรวจได้ที่:

```txt
Netlify project > Domain management > HTTPS
```

ถ้า HTTPS ยังไม่ผ่าน ให้ตรวจ:

- DNS record ชี้มาที่ Netlify ถูกต้องหรือไม่
- `www` เป็น CNAME ไปที่ `your-site-name.netlify.app` หรือไม่
- apex domain ชี้ตามค่าที่ Netlify แนะนำหรือไม่
- รอ DNS propagation ครบหรือยัง

## ปัญหาที่พบบ่อย

### 1. Deploy แล้วขึ้น Page not found

สาเหตุที่พบบ่อย:

- `Publish directory` ตั้งผิด
- ไม่มี `index.html` ใน publish directory

สำหรับโปรเจกต์นี้ให้ใช้:

```txt
Publish directory: .
```

### 2. CSS หรือ JavaScript ไม่โหลด

ตรวจว่า path ใน HTML เป็น path แบบ relative เช่น:

```html
<link rel="stylesheet" href="css/style.css">
<script src="js/app.js"></script>
```

ถ้าใช้ path ที่ขึ้นต้นด้วย `/` ต้องแน่ใจว่าไฟล์อยู่ตรงตำแหน่งนั้นจริงเมื่อ deploy

### 3. Push ไม่ได้เพราะ authentication

ให้ลอง:

- Login GitHub ใน browser ก่อน
- ใช้ GitHub Desktop
- ใช้ Git Credential Manager
- ใช้ personal access token เฉพาะเมื่อจำเป็น และห้ามเก็บ token ไว้ในไฟล์โปรเจกต์

### 4. Netlify หา repository ไม่เจอ

ให้ตรวจ:

- Netlify GitHub App มีสิทธิ์เข้าถึง repo แล้วหรือยัง
- Repo อยู่ใน account หรือ organization ที่เลือกถูกต้องหรือไม่
- ถ้า repo เป็น private ต้องให้สิทธิ์ Netlify เข้าถึง private repo

### 5. แก้ไฟล์แล้วเว็บไม่ update

ให้ตรวจ:

```powershell
git status
git log --oneline -5
git remote -v
git push
```

แล้วดูใน Netlify:

```txt
Project > Deploys
```

ถ้าไม่มี deploy ใหม่ แปลว่า push อาจยังไม่ถึง GitHub หรือ Netlify ยังไม่ได้เชื่อมกับ branch ที่ถูกต้อง

### 6. Firebase Login ขึ้น `auth/unauthorized-domain`

ให้เพิ่ม domain ของเว็บใน Firebase:

```txt
Firebase Console > Authentication > Settings > Authorized domains
```

เพิ่ม domain ที่ใช้งานจริง เช่น:

```txt
your-site-name.netlify.app
yourdomain.com
www.yourdomain.com
```

### 7. Firestore ขึ้น permission denied

ให้ตรวจ 4 จุดนี้:

- Login ด้วย Google สำเร็จแล้วหรือยัง
- เปิด Firestore Database แล้วหรือยัง
- Rules ตรงกับ path `users/{userId}/{document=**}` หรือไม่
- `projectId` ใน `js/config.js` ตรงกับ Firebase project ที่กำลังตั้งค่าหรือไม่

### 8. Google แจ้งว่า app ยังไม่ได้ verify หรือ access blocked

ให้ไปที่:

```txt
Google Cloud Console > APIs & Services > Google Auth platform
```

ตรวจว่า:

- เพิ่ม test users แล้ว ถ้า app ยังอยู่ใน Testing mode
- ใส่ privacy policy และ terms URL ถูกต้อง
- scopes ใน `Data Access` ตรงกับที่เว็บใช้
- ถ้าจะให้คนทั่วไปใช้ ต้อง publish app และทำขั้นตอน verification ตามที่ Google แจ้ง

### 9. Sync Sheets หรือ Calendar แล้วขึ้น scope ไม่พอ

ให้ลอง:

1. Logout จากเว็บ
2. Login ใหม่
3. ตอน Google ขอสิทธิ์ ให้กดยอมรับ scopes ทั้งหมด

และตรวจว่า `Google Auth platform > Data Access` มี scopes เหล่านี้:

```txt
https://www.googleapis.com/auth/spreadsheets
https://www.googleapis.com/auth/drive.file
https://www.googleapis.com/auth/calendar
```

### 10. Google API ยังไม่เปิดใช้

ถ้า error บอกว่า API ยังไม่ได้ enable ให้ไปที่:

```txt
Google Cloud Console > APIs & Services > Library
```

แล้ว enable:

```txt
Google Sheets API
Google Calendar API
Google Drive API
```
